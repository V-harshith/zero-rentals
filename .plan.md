# Owner Dashboard Implementation Plan

## Business Logic Decisions (CONFIRMED)

### 1. Plan Architecture (Two-Tier System)

**Tier 1: Subscription Plan** (Silver, Gold, Platinum, Elite)
- Determines FEATURES available to the owner
- Includes 1 property listing
- Features: Analytics access, max photos, support level, badges
- Stored in `subscriptions` table (one active per owner)

**Tier 2: Property Payment** (For additional properties beyond the first)
- Only needed for 2nd, 3rd, etc. properties
- Determines DURATION of the additional property
- DOES NOT change features - uses subscription plan features
- Stored in `properties` table columns

### 2. Free Plan Rules
- **Analytics:** Completely locked (no teaser, no views, nothing)
- **Properties:** Can post 1 property only
- **Photos:** Max 5 photos
- **Features:** No analytics, no featured badge, no priority support

### 3. Multiple Properties Scenario

**Example:** Owner has Silver plan + Property A (included)

**Wants to post Property B:**
- Must pay for additional property slot
- Can choose duration: 1mo (₹1000), 3mo (₹2700), 6mo (₹5000), 12mo (₹9000)
- Property B gets SAME features as subscription plan (Silver)
- Property B expires based on payment duration

**If they "upgrade" to Platinum:**
- Subscription plan changes to Platinum
- ALL properties (A and B) now get Platinum features
- Property B's expiry date doesn't change (still expires when paid duration ends)
- Profile badge shows "Platinum"

### 4. Profile Badge Logic
- Shows the active subscription plan name (Silver/Gold/Platinum/Elite)
- Only one badge - the current subscription tier
- Changing subscription updates the badge for all properties

### 5. Analytics Access Logic
```
if (subscriptionPlan === "FREE") {
  analytics = false;  // Complete lock
} else {
  analytics = true;   // All plans Silver+ get analytics
}
```
